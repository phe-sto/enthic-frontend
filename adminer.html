Base de données

La base de données peut ouvertement être explorée avec l'outil Adminer et le mot de passe ILoveData@2022.

Adminer permet de:

  • Executer des requêtes SQL.
  • Faire une recherche via l'interface sans requête SQL.
  • Exporter les résulats des recherches ou requêtes au format SQL, CSV, TSV, zippé ou non.
  • Tout ce que permet un logiciel d'administration de base de données complet.

La base de donnéess est constituée de 3 tables. La table identity stocke les données propres à l'identité de l'entrepise, le SIREN est donc l'identifiant unique. La table bundle stocke les liasses fiscales par compte et par année de déclaration. Il y a donc une relation one to many entre le SIREN de la table identity et le SIREN de la table bundle. La base request stocke les requêtes HTTP entrantes pour une analyse ultérieur.

Les colonnes ape, accountability et bundle stockent respectivement le code APE, le code correspondant au type de compte et le code de la liasse fiscale. Ces codes ne sont pas explicitent et sont dans les déclarations le plus souvent des chaînes de caractères. Par soucis d'efficacité de l'application, ils ont été remplacés par des TINYINT. La correspondance entre ces TINYINT, les codes des impôts, les codes APE et leur description peut se faire à l'aide de deux points d'API. http://api.enthic.fr/company/ontology pour les types de comptes, les liasses fiscales et http://api.enthic.fr/company/ape pour les codes APE.

N'hésitez pas à nous écrire par le biais du formulaire de contact pour toute aide au requêtage en base.
